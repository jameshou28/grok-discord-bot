# Grok Bot

A Discord LLM assistant powered by Gemini 2.5 Flash, with semantic knowledge retrieval over a local knowledge base.

## Key Features
- **RAG Knowledge Base:** Semantic search through your knowledge base using vector embeddings.
- **Smart Key Rotation:** Automatically cycles through multiple Gemini API keys to bypass quota limits, with a Hack Club AI fallback when all keys are exhausted.
- **Configurable persona:** The system prompt lives in a plain-text file you edit — no code changes.
- **Advanced Logging:** Dual-output logging to both terminal and `bot_logs.txt`.

## Repo layout
```
src/
  bot.py        Discord bot entrypoint
  embed.py      builds the embedding cache from knowledge.txt
data/
  knowledge.txt              your RAG knowledge source (gitignored — copy from .example)
  instructions.txt           your bot's system prompt / persona (gitignored — copy from .example)
  *.embed_cache.pkl          embedding cache, generated by embed.py (gitignored)
  knowledge.txt.example      template for knowledge.txt
  instructions.txt.example   template for instructions.txt
```

## Prerequisites
- **Python 3.9+**
- A Discord Bot Token (Message Content Intent enabled).
- One or more [Google Gemini API Keys](https://aistudio.google.com/app/apikey).

## Setup & Installation

**1. Install dependencies:**
```bash
python3 -m pip install -r requirements.txt
```

**2. Configure environment variables:**
```bash
cp .env.example .env
```
Open `.env` and fill in `DISCORD_TOKEN` and `GEMINI_API_KEYS` (comma-separated for multiple keys).

**3. Create your data files:**
```bash
cp data/knowledge.txt.example data/knowledge.txt        # your RAG content
cp data/instructions.txt.example data/instructions.txt  # your bot's persona (optional — has a built-in default)
```
Edit both to taste. `instructions.txt` sets how the bot behaves; if it's missing, `bot.py` falls back to a built-in default. (This is separate from the "About Me" text on the Discord profile, which is edited in the Discord Developer Portal.)

**4. Build the embedding cache:**
```bash
python3 src/embed.py
```
Re-run this whenever you edit `data/knowledge.txt` — the cache is keyed by file contents and won't rebuild on its own at bot startup.

## Running the Bot
```bash
python3 src/bot.py
```

## Usage
`@BotName <prompt>` — mention the bot anywhere it can read and it replies.
